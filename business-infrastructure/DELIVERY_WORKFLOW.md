# Delivery Workflow

This is the standard operating procedure for delivering the completed project to the client.

## Steps
1. **Verify Completion:** Ensure all intake fields have been mapped to the `config.ts` of the chosen template.
2. **Push to GitHub:** Push the completed codebase to a new public (or private, if agreed) GitHub repository.
3. **Deploy:** Connect the repository to Vercel or Netlify and deploy.
4. **Test Live URL:** Click all CTAs and ensure mobile responsiveness looks correct on the live domain.
5. **Record Loom:** Open the live URL and record a 2-3 minute Loom video (Script in `LOOM_SCRIPT.md`).
6. **Draft Email:** Copy the template from `CLIENT_DELIVERY_EMAIL.md`.
7. **Insert Links:** Add the Live URL, GitHub Repo link, and Loom video link to the email.
8. **Send Email:** Send the email and close the job tracker.

## Required Assets
- Live Vercel/Netlify URL
- GitHub Repository Link
- Loom Video Link
- Instructions for DNS (if applicable)
